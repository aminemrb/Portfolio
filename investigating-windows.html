Investigating Windows

🖥️ Windows Server 2016
🔍 Digital Forensics & Incident Response
🔗 tryhackme.com/room/investigatingwindows

// Contexte

Investigation forensique d'une machine Windows Server 2016 compromise. Objectif : reconstituer la chaîne d'attaque complète — accès initial, persistance, élévation de privilèges, C2 — en s'appuyant uniquement sur les outils natifs Windows et les journaux d'événements.

Étape 01

Version du système

▸Objectif Identifier précisément l'OS de la machine cible.
▸Méthode Exécution de systeminfo dans PowerShell.
▸Résultat Microsoft Windows Server 2016 Datacenter — Build 14393.
Étape 02

Dernier utilisateur connecté

▸Objectif Déterminer quel compte a ouvert la dernière session.
▸Méthode Filtrage de l'Observateur d'événements sur Event ID 4624 (ouverture de session réussie).
▸Résultat Administrator — dernier logon le 3/8/2026.
Étape 03

Dernière connexion de l'utilisateur John

▸Objectif Dater la dernière activité du compte John.
▸Méthode net user John | select-string "Last"
▸Résultat Dernière connexion le 3/2/2019 à 5:48:32 PM.
Étape 04

Persistance via le Registre

▸Objectif Identifier les programmes lancés automatiquement à chaque démarrage.
▸Méthode Inspection de la clé HKLM\...\CurrentVersion\Run dans regedit.
▸Résultat Entrée suspecte BadrClient exécutant un script pointant vers 10.34.2.3.
Étape 05

Analyse des privilèges administrateur

▸Objectif Lister tous les comptes disposant de droits admin locaux.
▸Méthode net localgroup administrators
▸Résultat Guest, Jenny
Étape 06

Tâche planifiée malveillante

▸Objectif Détecter les tâches planifiées utilisées pour la persistance.
▸Méthode Inspection du Planificateur de tâches (Task Scheduler) Windows.
▸Résultat Tâche anormale : Clean file system.
Étape 07

Analyse du payload — Reverse Shell

▸Objectif Identifier la charge utile exécutée par la tâche clean file system.
▸Méthode Lecture des Actions de la tâche dans le Planificateur.
▸Résultat Exécution de nc.ps1 -l 1348 — reverse shell PowerShell en écoute sur le port 1348.
Étape 08

Vérification de l'utilisateur Jenny

▸Objectif Vérifier si le compte Jenny a déjà été utilisé.
▸Méthode net user Jenny
▸Résultat Last Logon : Never — compte créé mais jamais utilisé.
Étape 09

Date de la compromission initiale

▸Objectif Dater précisément le début de la compromission.
▸Méthode Date de planification de la tâche.
▸Résultat L'attaque initiale remonte au 3/2/2019.
Étape 10

Élévation de privilèges

▸Objectif Identifier le moment exact où l'attaquant a obtenu des droits élevés.
▸Méthode Filtrage des logs de sécurité sur Event ID 4672 (attribution de privilèges spéciaux).
▸Résultat Privilèges spéciaux attribués à 4:04:49 PM le 3/2/2019.
Étape 11

Outil de vol d'identifiants — Mimikatz

▸Objectif Identifier les outils offensifs utilisés pour le vol de credentials.
▸Méthode Analyse des alertes Windows Defender — Event ID 1116 (détection malware).
▸Résultat Détection de HackTool:Win32/Mimikatz.l — dump de hachages NTLM en mémoire.
Étape 12

Serveur C2 — DNS Poisoning

▸Objectif Identifier l'infrastructure Command & Control de l'attaquant.
▸Méthode Analyse du fichier C:\Windows\System32\drivers\etc\hosts.
▸Résultat google.com redirigé vers 76.32.97.132 — IP du serveur C2 de l'attaquant.
Étape 13

Découverte du Web Shell

▸Objectif Identifier le vecteur d'accès initial au serveur web.
▸Méthode Inspection du répertoire IIS : C:\inetpub\wwwroot.
▸Résultat Fichier b.jsp détecté — webshell permettant l'exécution de commandes via HTTP.
restreint
Étape 14

Ouverture de port dans le Pare-feu

▸Objectif Détecter les règles de pare-feu ajoutées par l'attaquant.
▸Méthode Analyse des règles Inbound dans le Pare-feu Windows Defender avec fonctions avancées.
▸Résultat Règle frauduleuse "Allow outside connections for development" autorisant le trafic sur le port TCP 1337.

// Synthèse — Chaîne d'attaque

1. Accès Initial (T1505.003) : L'intrusion commence par la compromission du serveur web où l'attaquant parvient à déposer un webshell.

2. Vol d'Identifiants (T1003) : Une fois l'accès initial sécurisé, l'outil Mimikatz est déployé sur le système pour extraire les hachages NTLM ou les mots de passe en clair directement depuis la mémoire du serveur.

3. Élévation de Privilèges (Event 4672) : L'attaquant consolide son emprise en obtenant les droits d'administration. Les journaux révèlent l'attribution de privilèges spéciaux

4. Persistance Multicouche (T1547.001 & T1053.005) : Pour survivre à un redémarrage, l'attaquant ancre sa présence à deux niveaux : la création d'une clé de registre BadrClient qui s'exécute automatiquement, et la mise en place de la tâche planifiée clean file system lançant un reverse shell via nc.ps1 sur le port 1348.

5. Command & Control (T1071) : La communication avec l'infrastructure pirate est garantie par un DNS poisoning redirigeant le trafic légitime vers l'IP 76.32.97.132. Pour parfaire son contrôle, une règle de pare-feu est ajoutée pour ouvrir le port TCP 1337 aux connexions externes.
